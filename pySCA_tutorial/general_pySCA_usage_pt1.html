The annotateMSA script provides utilities to automatically annotate sequence headers (for a FASTA file) with taxonomic information. Currently this can be done in one of two ways:¶

1) For Pfam alignments, annotations can be extracted from the file
   pfamseq.txt (please download from:
   ftp://ftp.ebi.ac.uk/pub/databases/Pfam/current_release/database_files/pfamseq.txt.gz

2) For Blast alignments, annotations can be added using the NCBI Entrez
   utilities provided by BioPython. They can be based on GI or accession
   numbers that are used to query NCBI for taxonomy information (note that
   this approach requires a network connection).

To extract GI or accession numbers, use the scripts alnParseGI.py or alnParseAcc.py, respectively.

For both the Pfam and NCBI utilities, the process of sequence annotation can be slow (on the order of hours, particularly for NCBI entrez with larger alignments). However, the annotation process only needs to be run once per alignment.

The scaProcessMSA script conducts the basic steps in multiple sequence alignment (MSA) pre-processing for SCA, and stores the results using the python tool pickle:¶

1) Trim the alignment, either by truncating to a reference sequence (specified with the -t flag) or by removing excessively gapped positions (set to positions with more than 40% gaps)

2) Identify/designate a reference sequence in the alignment, and create a mapping of the alignment numberings to position numberings for the reference sequence.

The reference sequence can be specified in one of
four ways:

    a) By supplying a PDB file - in this case, the reference sequence
       is taken from the PDB (see the pdb kwarg)

    b) By supplying a reference sequence directly (as a fasta file -
       see the refseq kwarg)

    c) By supplying the index of the reference sequence in the
       alignment (see the refseq kwarg)

    d) If no reference sequence is supplied by the user, one is
       automatically selected using the scaTools function chooseRef.

The position numbers (for mapping the alignment) can be specified in
one of three ways:

    a) By supplying a PDB file - in this case the alignment positions
       are mapped to structure positions

    b) By supplying a list of reference positions (see the refpos
       kwarg)

    c) If no reference positions are supplied by the user, sequential
       numbering (starting at 1) is assumed.

3) Filter sequences to remove highly gapped sequences, and sequences with an identity below or above some minimum or maximum value to the reference sequence (see the parameters kwarg) 4) Filter positions to remove highly gapped positions (default 20% gaps, can also be set using --parameters) 5) Calculate sequence weights and write out the final alignment and other variables

This dictionary stores the following:

$Dseq[key] = values$

  • alg = list containing the amino acid sequences in the processed alignment
  • hd = species annotation for each sequence in the processed alignment
  • msa_num = The alignment as an M x N matrix, M is the number of sequences, N is the number of positions. Each amino acid position is represented as a number instead of its amino acid letter.
    • 'ACDEFGHIKLMNPQRSTVWY' = '1 2 3 4 5 6 ...20'
    • example: 'MISL' = [10, 7, 15, 9]
  • seqw = each sequence in the alignment is down-weighted based on its similarity to the reference. This stores the weight for each sequence relative to the reference.
  • Nseq = number of sequences in the MSA
  • Npos = number of amino acid positions in the alignment
  • ats = contains index of each amino acid position to its position in the reference sequence.
  • effseqs = number of total sequences after applying weight (seqw). The more similar the sequence is to the reference, the smaller its contribution to the number of effective sequences.
  • limitseqs = A boolean that tells you if you sub-sampled the MSA. Default = 'False'
  • *NseqPrelimit = number of sequences in the alignment after filtering
  • *effseqsPrelimit = number of effective sequences in the alignment after filtering
  • pdbid = PDB ID (stored your input argument to scaProcessMSA)
  • pdb_chainID = PDB chain ID (was an input argument to scaProcessMSA)
  • **distmat = a matrix that is Npos x Npos. Pair-wise distance of amino acid positions in the reference sequence (after filtering out gaps in the sequence)
  • i_ref = index of your reference sequence in the alignment. To get pull out this sequence, enter $Dseq['aln']['iref']$. To pull out the species annotation, enter $Dseq['hb']['iref']$.
  • trim_parameters = stored input argument PARAMETERS to scaProcessMSA. They set the thresholds for sequence identity and gaps to the reference sequence for a given sequence. See the cell above for more detailed description of each parameter.
  • truncate_flag = a boolean that tells you if you decided to truncate the alignment to the positions in your reference sequence. Default = 'False'

Applying the core SCA calculations to the processed MSA¶

The scaCore script runs the core calculations for SCA, and stores the output using the Python tool pickle. These calculations can be divided into two parts:¶

1) Sequence correlations:

  1. Compute simMat = the global sequence similarity matrix for the alignment

  2. Compute Useq and Uica = the eigenvectors (and independent components) for the following sequence correlation matrices:

    1. unweighted ($U^{0}$)
    2. sequence weights applied ($U^{1}$)
    3. both sequence and position weights applied ($U^{2}$)

2) Positional correlations:

  1. Compute the single-site position weights and positional conservation values ($D_{i}$ and $D_{i}^{a}$)
  2. Compute the dimension-reduced SCA correlation matrix $\tilde{C}_{ij}$, the projected alignment $tX$, and the projector
  3. Compute Ntrials of the randomized SCA matrix, and the eigenvectors and eigenvalues associated with each

What are the variables stored in Dsca? Here as a reference. These data will be analyzed more closely in part II.¶

Dia = evolutionary convservation for an amino acid, a, at position, i, in the reference sequence

Di = overall positional evolutionary conservation

Csca = $\tilde{C}_{ij}$ = the SCA matrix of pair-wise co-evolution between each pair of positions in the sequence, i and j.

tX = $x$ = The multiple sequence alignment represented as a projected matrix (M sequences x N positions).

simMat = pair-wise sequence similarity matrix (M sequences x M sequences)

Ntrials = number of times the sca matrix is randomized to evaluate significance of co-evolution signal.

Crand = $\tilde{C}_{rand}$ =randomized SCA matrix (Csca) that removes positional correlations.

Vrand = $\tilde{V}_{rand}$ = eigenvalues from Crand eigendecomposition of Crand

Lrand = $\tilde{\Delta}_{rand}$ = eigenvectors from Crand eigendecomposition of Crand

Useq = $U$ = Sequence correlations calculated by SVD using the matrix representation of the MSA

Uica =Independent components analysis of Useq to maximize independence between vectors.

Proj = projector used to reduce the MSA matrix, $X$ (M sequences x Npositions x 20 amino acids), to $x$ (M sequences x N positions)

lbda = $\lambda$ = a pseudocounter for matrix regularization

Before proceeding with analzying the results of SCA, let's take a second to evaluate the quality of the MSA.¶

Follow these three guidelines to see if your alignment is appropriate for analysis with SCA¶

  1. The number of effective sequences should be greater than or equal to 100.
  1. The alignment should be fairly diverse. The mean sequence identity should be between 20-40%. This is an empirical value. SCA will not work if sequence diversity is too high and if there is no sequence diversity in the alignment (what does it mean for SCA to not work? Does high sequence diversity mean that all of the positions in the entire protein get ID-ed as a sector (non-sparse signal) and that low sequence diversity measn that no positions in the sequence are ID-ed in the sector (too sparse signal)?
  1. The positional conservation across the sequence of your protein of interest should have regions of low and high conservation. With positions that are known to be essential to function have relatively higher conservation. For example, D27 in E. coli DHFR is essential to the redox reaction it catalyzes - its conservation is >3.

A cyan line at $D_{i} = 0.5$ marks an empirical conservation threshold.

  • A sufficiently diverse alignment contains positions above and below this level of conservation.
  • No positions below threshold -> An alignment that is too conserved -> SCA signal is not sparse enough
  • Positions above threshold -> An alignment that is not sufficiently conserved -> SCA signal is too sparse

Decomposition of the SCA matrix using scaSectorID¶

The scaSectorID script does the preliminaries of sector identification and stores the outputs using the python tool pickle:

1) Chooses $k_{max}$ (the number of significant eigenmodes) by comparison of the $C_{ij}$ eigenspectrum to that for the randomized matrices

2) Rotates the top $k_{max}$ eigenvectors using independent components analysis

3) Defines the amino acid positions that significantly contribute to each of the independent components (ICs) by empirically fitting each IC to the t-distribution and selecting positions with greater than a specified cutoff (default: p=0.95) on the CDF.

4) Assign positions into groups based on the independent component with which it has the greatest degree of co-evolution.

What are the variables stored in Dsect?¶

Eigendecomposition of the SCA matrix¶

Vsca = $\tilde{V}$ =eigenvectors from eigenvalue decomposition of Csca.

Lsca =$\tilde{\Delta}$ = eigenvalues from eignevalue decomposition of Csca.

Independent components analysis of eigenvectors¶

Vpica = $\tilde{V}^P_{1...k*}$ = Vsca after Independent Components Analysis (ICA)

Wpica = $W$ = transformation matrix to convert from Vsca to Vpica. Uesd in ICA to maximize the independence between eigenvectors.

ics = independent components and amino acid positions in each one

Determining significant Independent Components¶

kpos = $k*$ = the number of significant eigenmodes from eigenvaule decomposition of Csca. Calculated from sca.choosekpos(Lsca,Lrand). This takes eigenvalues from Csca decomposition and eigenvalues from decomposition of randomized Csca matricies (Crand) and computes the number of significant eigenmodes.

scaled_pd = probability t-distributions of co-evolution signal of all positions within each significant IC.

pd = parameters of scaled_pd (mean and standard deviation)

cutoff = significance threshold. Defines significant co-evolving amino acid positions in each IC.

sortedpos = from most to least significant signal for each IC.

icsize = number of amino acid positions in each significant IC

Determining correlations between sequences that contribute to positional correlations found in significant ICs¶

Usca = $\tilde{U}$ = Eigenvectors of sequence correlation matrix. Directly related to the positional correlation matrix by SVD.

Upica = $\tilde{U}^P$ = $W\tilde{U}$ = ICA on Usca with transformation matrix, Wpica ($W$)
